# 🛡️ AutoClaw (澳龍) 安全使用手冊

## 前言：理解「龍蝦」的權限
AutoClaw 不同於一般的聊天機器人（如 ChatGPT 網頁版）。它是一個 **Agent**，意味著它具備「手」和「腳」：它可以執行 Shell 指令、讀寫您的硬碟檔案、操作瀏覽器，甚至是代表您在網路平台上進行操作。
**權限越高，風險越大。**

---

## 一、 核心安全原則：「三不、三要」

### ❌ 三不原則
1.  **不脫離沙盒執行**：除非必要，否則不要隨意修改澳龍的「沙盒 (Sandbox)」設定，這層防護牆確保了 Agent 萬一出錯時，不會波及到您的整個作業系統。
2.  **不餵食極端敏感數據**：雖然澳龍支援本地化運行，但部分 Skills 仍需調用雲端 API。請勿在未確認路徑前，將公司核心源碼、薪資單或極度私密的個人身分證件直接餵給 Agent 處理。
3.  **不授權高風險自動化**：避免在未經人工審核的情況下，授權 Agent 自主執行「刪除檔案」、「重設伺服器」或「發送對外付款指令」等不可逆動作。

### ✅ 三要原則
1.  **要人工在場 (Human-in-the-loop)**：特別是當 Agent 使用 `exec` 工具執行腳本時，務必觀察它的輸出紀錄。
2.  **要定期清理憑證**：澳龍中會存儲許多 API Key (如 Tavily, Google Workspace 等)，建議每個月更新一次密鑰，並使用 `secrets` 管理工具進行加密。
3.  **要使用權限最小化原則**：只給予 Agent 完成當下任務所需的最低權限。

---

## 二、 技術防護：特定工具的安全策略

### 1. `exec` (指令執行) 安全
*   **風險**：Agent 可能因為幻覺執行了 `rm -rf /` (格式化硬碟) 或下載不明病毒。
*   **防範**：
    *   澳龍內建了**黑名單機制**，預設禁止執行高風險系統命令。
    *   在執行任何複雜腳本前，請讓 Agent 先解釋腳本內容。

### 2. `browser` (瀏覽器操作) 安全
*   **風險**：自動點擊不明廣告連結，或意外洩漏您已登入的 Cookie 資訊。
*   **防範**：
    *   使用澳龍提供的 **專屬隔離瀏覽器**，不要直接操作您正在登入私人帳號（如銀行、FB）的日常瀏覽器。
    *   關閉瀏覽器的「自動填寫密碼」功能。

### 3. `filesystem` (檔案系統) 安全
*   **風險**：Agent 誤刪重要文件。
*   **防範**：
    *   **路徑限制**：將 Agent 的工作空間限制在特定的資料夾內，禁止其越權訪問系統核心路徑。
    *   **定期備份**：確保您的重要資料有版本控制或異地備份。

---

## 三、 針對「智譜 AI」特點的進階建議

1.  **啟用 Claw for Enterprise Security**：如果您是在企業環境中使用，建議開啟智譜提供的企業級安全管理體系，它具備更強的多層防護與追蹤能力。
2.  **模型一致性檢查**：在使用 GLM-5-Turbo 等強大模型時，若發現 Agent 開始反覆嘗試錯誤的操作，請立即終止該 Session (使用 `openclaw sessions kill`)。
3.  **注意 API 額度異常**：如果您發現 Token 消耗突然暴增，可能是 Agent 在背景執行了大量不必要的重複搜尋。

---

## 四、 應急預案：當 Agent 出現異常時
*   **第一步：Ctrl+C** → 在終端機強行終止當前任務。
*   **第二步：斷網** → 如果懷疑 Agent 正在進行異常數據傳輸，立即斷開網路連接。
*   **第三步：重設配置** → 使用 `openclaw reset` 清除可能被惡意篡改的本地狀態。
